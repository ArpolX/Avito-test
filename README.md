# Тестовое задание авито
# Запуск
Клонируйте репозиторий.
```
git clone https://github.com/ArpolX/Avito-test
```
Используйте `docker compose up --build` или `./build.sh` для запуска веб-сервера. Он будет слушать стандартный `8080` порт.
# Небольшое описание
Я реализовал все обязательные и несколько дополнительных задания.
## Два дополнительных эндпоинта
* /amount/prOpen - показывает количество PR, которые находятся в статусе Open. На вход ничего не принимает.
* /team/switchFalse - меняет активность всей команды на false. Принимает:
```
{
    "team_name": "название команды"
}
```
## Нагрузочное тестирование
Провёл нагрузочное тестирование с помощью hey(я думал в данном случае, когда средняя rps=5 и количество команд не доходит до 100, такого решения будет достаточно). Результаты лежат в файле `load_test_results.txt`.
## Описание линтера
Я использовал `golangci-lint`. Основной конфиг находится в `.golangci.yml` и включает следующие проверки:

- **govet** — стандартные ошибки Go
- **errcheck** — контроль игнорируемых ошибок
- **staticcheck** — расширенный статический анализ
- **gosimple** — упрощение кода
- **unused** — поиск неиспользуемых переменных/функций
- **gocyclo** — цикломатическая сложность функций
- **gofmt** — проверка форматирования
- **revive** — проверка стиля кода

**Настройки:**

- Сложность функции (`gocyclo`) ≥ 15 вызывает предупреждение.
- `gofmt` упрощает выражения автоматически.
- `revive` использует отдельный конфиг `revive.toml` для правил стиля.
